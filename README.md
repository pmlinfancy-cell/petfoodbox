## 🎯 为什么需要猫食盒子？—— 痛点分析
在校园和社区的流浪猫救助中，传统投喂方式存在两大核心痛点：

| 痛点 | 具体表现 |
|------|----------|
| **生存环境恶劣** | 区域性野猫食物来源不稳定，依赖零散投喂，长期处于饥饿或营养不良状态 |
| **健康与生命威胁** | 缺乏科学喂养，病死率高；野外进食易引发中毒；无实时监控导致伤病无法及时救治 |

**猫食盒子** 是一套面向校园与社区的 **户外智能投食系统**，
由 **硬件终端 + 小程序前端 + 后台管理系统** 构成。  
用户通过小程序选择投喂设备、支付少量费用（如1元/20mg猫粮），即可远程触发户外投食机出粮，并通过 **猫脸识别** 记录是哪只猫吃到了猫粮，形成可追踪、可复盘、有情感连接的投喂闭环。

项目源于深大校园记忆，依托深大黑客松发起，希望用技术改善流浪猫的生存现状，同时维护校园生态平衡。

## ✨ 新增功能亮点 —— 我们如何解决

相比普通投食器或人工投喂，本项目新增以下核心能力：

### 1. 赛博投喂 · 全程可信
- **每笔订单可追踪**：用户支付后，实时查看状态 `队列中 → 已出粮 → 已进食`
- **视频/图片回传**：硬件自动拍摄进食过程，投喂证据一目了然
- **订单与猫粮批次绑定**：猫粮来源、批次、临期时间全部可查

### 2. 猫脸识别 · 建立猫咪档案
- 基于开源模型（如 sysucats/cat-face）实现猫脸识别
- 每只猫拥有**独立电子档案**（名字、年龄预估、品种、投喂历史）
- 支持用户给猫**命名**，增强情感连接与留存
  
### 3. 低门槛运营闭环
- 运营后台可统一管理：设备、订单、猫粮批次、识别库、视频
- 支持人工审核新猫入库 + 高置信自动入库

### 4. 从深大出发，可复制到更多场景
- 当前 Demo 覆盖深大校园地图，后续可扩展多校区、社区、救助点

## ✨ 核心功能

### 用户端（小程序 / Demo）

- 🗺️ **校园地图**：展示深大校园内设备点位、在线状态、投喂入口
- 💰 **极速投喂**：3步内完成支付（1元/20mg起），支持套餐选择
- 📹 **投喂记录**：查看订单状态（队列中 → 已出粮 → 已进食），回传视频/截图
- 🐱 **猫猫档案**：为识别到的猫咪命名、查看年龄/品种/近期投喂历史
- 🔐 **微信授权登录**：保存个人投喂记录与猫咪互动

### 硬件端

- 🌧️ **户外耐用**：防雨、防晒、防咬、防尘设计
- ⚙️ **精准出粮**：按订单量自动出粮，支持远程控制与自检
- 📡 **视频回传**：每次投喂生成短视频或图片证据
- 🔄 **状态同步**：在线/离线监控，断网重连与补偿队列

### 后台管理系统

- 📦 **订单管理**：全订单状态流转、异常处理、退款
- 🐈 **猫脸训练库**：管理样本、识别版本、人工审核入库
- 🥣 **猫粮批次**：库存、临期提醒、更换记录
- 🛠️ **设备管理**：设备列表、固件版本、远程维护
- 📊 **运营配置**：套餐开关、轮播图、地图点位

---

## 🧠 技术亮点

- **猫脸识别**：基于开源方案（如 sysucats/cat-face）自训练模型，高置信自动入库 + 低置信人工审核
- **状态机驱动**：`已创建 → 已出粮 → 已进食`，任意阶段可转入异常并通知运营
- **人机协同闭环**：动态调控投喂量，避免过度聚集或生态失衡
- **模块化解耦**：前端可先运行 HTML Demo，后续平滑迁移至小程序与真实支付
